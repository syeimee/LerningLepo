# 学習テーマ
作業日時: 2025-06-04


## 目的・背景 
Javaでコンソールのアプリケーションを開発する


## 実装内容・学んだ技術  
クラスの定義、インスタンシエイトの仕方



## 課題・問題点  
- クラスの役割を明確にする。Productクラスに対してProductManagerクラスはわかりにくい
なんちゃらControllerとかなんちゃらserviceの方がいいのではないか

- バリデーションの実装箇所と標準入力の箇所が同じ箇所にあるので、
保守性が悪い。インスタンス化する前に実装するのかした方がいい



## 気づき・改善案  
- 責務に関して
全体の構成として、

```
Main（アプリの流れ）
├── InputHandler（入力処理）
├── ProductRepository（商品保存・一覧）
├── Product（モデル）
```

がいいのではないか？

バリデーションを行うのは、入力バリデーションとして、InputHandler（入力処理）に責務を持たせるのか
ビジネスバリデーションとしてProduct（モデル）に責務を持たせるのかを吟味する

- 商品コードは標準入力で受け付ける


- Clean Architecture読んでみる？
