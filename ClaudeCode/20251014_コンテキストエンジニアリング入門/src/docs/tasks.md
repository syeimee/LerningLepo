### 技術的な実装詳細
- **状態管理**: React Context API（SessionContext, DataContext）
- **フォーム**: React Hook Form + Zod
- **UI**: Material-UI (MUI) v5
- **データフロー**: 日報作成 → 授業ステータス自動更新（COMPLETED）
- **ロールベースアクセス**: ADMIN（全表示）、TEACHER（担当分のみ）、STUDENT（自分のみ）
- **データ取得**: Server Component（Prisma） + Client Component（UI/インタラクション）
- **キャッシュ戦略**: revalidatePath による Server Component の再検証
- **検索・フィルタリング**: useMemo による効率的なフィルタリング処理

### 注意事項・メモ
- Supabaseの無料枠: 500MB、10,000行まで
- Google Sheets APIのレート制限に注意
- 日報作成時に授業ステータスを自動更新する仕様（Prismaトランザクションで実装）
- Server ActionでCRUD操作後は必ず`revalidatePath()`を呼び出してキャッシュを更新
- Client ComponentでpropsとしてServer Componentからデータを受け取る場合：
  - 状態管理が不要な場合はpropsを直接使用（例: DailyReportsList）
  - 状態管理が必要な場合は`useEffect`でpropsの変更を監視（例: MonthlyReportsList）
- Next.js 14のApp Routerでは、Server ComponentとClient Componentの適切な分離が重要
