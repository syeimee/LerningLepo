## 1. 概要
本アプリは、Spring Frameworkを用いたREST APIベースのECサイトWebサービスとして設計され、**コンテンツサーバ (Conoha WING)** と **APIサーバ (Conoha VPS)** を分離し、拡張性の高いアーキテクチャを採用する。

## 2. 技術スタック
| コンポーネント | 使用技術 |
|--------------|------------------|
| フロントエンド | React または Vue |
| バックエンド | Java 21 + Spring Boot |
| データベース | MySQL + Spring Data JPA |
| 認証 | Spring Security (JWT) |
| 決済サービス | Stripe API |
| インフラ | Conoha WING (コンテンツ), Conoha VPS (API) |
| バックアップ | その他VPSによる定期バックアップ |

## 3. システム構成
### 3.1 APIサーバ (Spring Boot)
- **Spring Bootを使用し、マイクロサービス化可能なアーキテクチャを構築**
- **Spring SecurityでJWT認証を実装**
- **Spring Data JPAでMySQLとの連携**
- **Spring WebFluxで非同期処理を導入**

### 3.2 認証・セキュリティ
- **APIの認証:** Spring Security + JWT（Bearerトークン）
- **通信の暗号化:** HTTPS（SSL証明書）
- **ユーザー権限管理:** ロールベースアクセス制御

### 3.3 データベース設計
- **ORM:** Spring Data JPA  
- **キャッシュ:** Redisを使用し、頻繁にアクセスされるデータを高速化  
- **バックアップ:** MySQLの定期バックアップ

## 4. 運用・メンテナンス
### 4.1 ログ監視
| 項目 | 監視方法 |
|------|---------|
| APIログ | ELK Stack（Elasticsearch, Logstash, Kibana） |
| DB監視 | MySQL Query Logでパフォーマンス解析 |

ELK StackはBasic(無料版)を使用する。

運用・監視については、エラー発生時に原因分析を目的として実施するが、常時監視は行わない。

### 4.2 デプロイ戦略
- **GitHub Actionsを使用しCI/CDを構築**
- **Dockerコンテナ化し、本番環境へデプロイ**

### 4.3 バックアップ
- **サーバー間のバックアップは1日1回差分取得とする**
- **VPSのメンテナンス等によるバックアップ漏れは保証しない**
- **DBがバックアップするテーブルは、トランデータのみとする**

## 5. 開発手法
本PJに参加するメンバは本業があるため、確実な工数の確保ができない。
また、前回のPJのようにエター化を防ぐため、アジャイル＋ウォーターフォールの手法で開発を行う。

- スプリントは1ヶ月〜2ヶ月
- バックログにNotionを利用する
- AIを使用して各資料等の効率的な作成を行う
- ユーザーの利用開始まで検証サーバは作成せず、本番環境内でURIを分けて環境作成を行う
- 設計→製造→テスト→リリースの工程は、前工程が終わるまで基本的に開始しない
- 資料のレビューはPMまでとし、顧客レビューは行わない。
