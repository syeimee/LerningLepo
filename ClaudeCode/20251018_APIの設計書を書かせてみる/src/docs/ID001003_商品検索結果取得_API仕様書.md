## 2.概要

ECサイトの商品検索機能で使用するAPI。
商品名の部分一致検索により商品情報を取得し、ページネーションに対応する。

## 3.パラメータ

### 3.1.URI

`/products/search/get`

[API一覧 2. API一覧 参照](./API一覧.md)

### 3.2.インプット

```json
{
  "searchKeyword": "ぶどう",
  "limit": 20,
  "offset": 0
}
```

| パラメータ名 | 型 | 必須 | 説明 |
|------------|-----|------|------|
| searchKeyword | string | 必須 | 検索キーワード。商品名の部分一致検索に使用 |
| limit | number | 任意 | 取得件数（デフォルト：20、最大：100） |
| offset | number | 任意 | 取得開始位置（デフォルト：0） |

### 3.3.アウトプット

```json
{
  "products": [
    {
      "productId": "p00000000001",
```

## 5.処理詳細

### 5.1 バリデーションチェック
1. インプットの定義通りかバリデーションチェックを行う。
   1. searchKeywordが文字列型であることを確認する。
   2. searchKeywordが空文字でないことを確認する。
   3. limitが指定されている場合、数値型で1〜100の範囲内であることを確認する。
   4. offsetが指定されている場合、数値型で0以上であることを確認する。
   5. **定義通りでないインプットがあった場合、処理を中断する**
      1. エラーログ(E001-0003-0001)を出力する。
      2. エラー(E001-0003-0001)を返却する。

### 5.2 商品検索結果の取得
1. 「商品検索結果」を取得する。[8.1.商品検索結果取得](#81商品検索結果取得)
   1. **エラーが発生した場合、処理を中断する**
      1. エラーログ(E001-0003-9902)を出力する。
      2. エラー(E001-0003-9902)を返却する。
2. 取得した「商品検索結果」を「商品リスト」に格納する。
3. 検索結果の全商品件数を「全件数」に格納する。

### 5.3 レスポンスの返却
1. 以下のレスポンスパラメータを設定し、返却する。

| レスポンスパラメータ | 設定値 |
|-------------------|--------|
| products | 「商品リスト」 |
| total | 「全件数」 |
| searchKeyword | インプットのsearchKeyword |

## 8.SQL

### 8.1.商品検索結果取得

```sql
-- 商品検索結果取得
SELECT
  p.product_id,
  p.description as product_name,
  p.description,
  p.price,
  p.stock_quantity,
  p.category_id,
  c.name as category_name,
  p.producer_id,
  pr.name as producer_name,
  pi.image_path,
  pi.view_order
FROM PRODUCT p
INNER JOIN CATEGORY c ON p.category_id = c.category_id
LEFT JOIN PRODUCER pr ON p.producer_id = pr.producer_id
LEFT JOIN PRODUCT_IMAGE pi ON p.product_id = pi.product_id
WHERE p.disabled = 0 -- 有効な商品のみ
  AND c.disabled = 0 -- 有効なカテゴリのみ
  AND (pr.disabled = 0 OR pr.disabled IS NULL) -- 有効な生産者のみ
  AND (pi.disabled = 0 OR pi.disabled IS NULL) -- 有効な画像のみ
  AND p.name LIKE CONCAT('%', ?, '%') -- 商品名部分一致検索
ORDER BY p.created_at DESC, pi.view_order ASC
LIMIT ? OFFSET ?;

-- 全件数取得
SELECT COUNT(DISTINCT p.product_id) as total
FROM PRODUCT p
INNER JOIN CATEGORY c ON p.category_id = c.category_id
WHERE p.disabled = 0 -- 有効な商品のみ
  AND c.disabled = 0 -- 有効なカテゴリのみ
  AND p.name LIKE CONCAT('%', ?, '%'); -- 商品名部分一致検索
```

## 9.備考

- 商品名（PRODUCT.name）の部分一致検索を行う
- 検索キーワードは前方一致・後方一致・部分一致すべてに対応する（LIKE '%keyword%'）
- 商品画像は複数登録されている可能性があるため、view_orderでソートして取得する
- limitのデフォルト値は20、最大値は100とする
- 削除フラグ(disabled)が1の商品、カテゴリは取得対象外とする
- 商品の並び順は作成日時の降順（新しい順）とする
- 検索結果が0件の場合でも正常応答とし、空配列を返却する
- SQLのパラメータバインディングには`?`を使用する（MariaDB準拠）
