# 幼児向けプログラミング教育アプリ 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
**KidsCoding** (仮称)

### 1.2 目的
3-4歳の幼児を対象とした、ビジュアルプログラミング教育ウェブアプリケーションの開発。
文字が読めない幼児でも直感的に操作でき、プログラミング的思考の基礎を楽しく学べる環境を提供する。

### 1.3 差別化ポイント
- **対象年齢**: 3-4歳に特化（5歳以上向けの既存サービスとの差別化）
- **文字を使わないUI**: 完全にアイコンベース、音声ガイド付き
- **親子学習の促進**: 保護者と一緒に楽しめる設計

---

## 2. ペルソナ定義

### 2.1 主要ペルソナ（利用児童）

**名前**: たけるくん
**年齢**: 4歳
**特徴**:
- ひらがなはまだ読めない
- タブレット操作には慣れている（動画視聴など）
- 動物や乗り物が大好き
- 達成感を得ると喜ぶ
- 集中力は10-15分程度

**学習目標**:
- 順序立てて考える力
- 原因と結果の関係性の理解
- 試行錯誤する楽しさを知る

### 2.2 サブペルソナ（保護者）

**名前**: お母さん（30代）
**特徴**:
- 子供の教育に関心が高い
- プログラミング教育の重要性は認識しているが、自身は詳しくない
- 子供と一緒に学びたい
- タブレットやPCは日常的に使用

**ニーズ**:
- 安全で適切な教材
- 子供の進捗が分かる機能
- 親子で楽しめるコンテンツ

## 3. 機能要件

### 3.1 コア機能

#### 3.1.1 ビジュアルプログラミングエディタ
- **命令ブロック（パネル）のドラッグ&ドロップ**
  - タッチ操作に最適化された大きなUI
  - 直感的なドラッグ操作

- **命令の種類（段階的導入）**

  **レベル1（初期）**: 基本的な方向指示
  - 前に進む
  - 後ろに進む
  - 左に曲がる
  - 右に曲がる

  **レベル2（中級）**: ループ（繰り返し）
  - 指定回数繰り返す（2回、3回など視覚的に表示）

  **レベル3（上級）**: 条件分岐
  - 障害物があったら〜
  - ゴールに着いたら〜

#### 3.1.2 3Dキャラクター制御
- **gLTFモデルの表示と操作**
  - Three.jsを使用した3D表示
  - スムーズなアニメーション

- **複数のキャラクター/オブジェクト**
  - 動物（犬、猫、うさぎなど）
  - 乗り物（車、電車、ロケットなど）
  - ユーザーが選択可能

#### 3.1.3 ステージ・ミッション制
- **段階的な学習コンテンツ**
  - チュートリアルステージ（保護者ガイド付き）
  - 基本ステージ（方向指示のみ）
  - 中級ステージ（ループを使う）
  - 上級ステージ（条件分岐を使う）

- **クリア条件**
  - キャラクターを特定の場所まで移動させる
  - アイテムを集める
  - 障害物を避ける

#### 3.1.4 フィードバックシステム
- **成功時**
  - 楽しい効果音・音楽
  - キャラクターの喜びアニメーション
  - キラキラエフェクト
  - 音声メッセージ「やったね！」「すごい！」

- **失敗時**
  - 優しい効果音
  - キャラクターの残念アニメーション
  - 「もう一回やってみよう！」という励まし

### 3.2 補助機能

#### 3.2.1 音声ガイド
- 全ての操作に音声説明を付与
- 子供向けの優しい声での案内
- 多言語対応（将来的に）

#### 3.2.2 進捗管理
- ステージのクリア状況を保存
- 獲得したスター/メダルの表示
- 保護者向けの進捗ダッシュボード

#### 3.2.3 カスタマイズ機能
- キャラクターの選択
- ステージの背景選択
- 難易度調整（親が設定）

## 4. UI/UX要件

### 4.1 デザイン原則

#### 4.1.1 文字を使わない設計
- すべての操作をアイコンで表現
- カラフルで視覚的に分かりやすいデザイン
- 音声ガイドで補完

#### 4.1.2 大きなタッチターゲット
- ボタンサイズ: 最小80x80px
- 十分な余白を確保
- 誤タップを防ぐレイアウト

#### 4.1.3 直感的なフィードバック
- タッチ時の視覚的反応（拡大、色変化）
- 触覚フィードバック（バイブレーション）
- 音声フィードバック

### 4.2 画面構成

#### 4.2.1 メイン画面
```
┌─────────────────────────────────────┐
│  [ホーム] [設定] [ヘルプ]           │  ← ヘッダー
├─────────────────────────────────────┤
│                                     │
│         3Dビューポート              │  ← キャラクター表示
│      (キャラクターとステージ)        │
│                                     │
├─────────────────────────────────────┤
│  [▲] [▼] [◄] [►] [🔁] [⚡]       │  ← 命令パネル
├─────────────────────────────────────┤
│  プログラム実行エリア               │  ← ドロップゾーン
│  [ここにブロックを置いてね]          │
├─────────────────────────────────────┤
│  [実行] [リセット] [ヒント]         │  ← アクションボタン
└─────────────────────────────────────┘
```

#### 4.2.2 ステージ選択画面
- 絵本のような見た目
- 各ステージが大きなアイコンで表示
- クリア済みにはスターマーク

#### 4.2.3 キャラクター選択画面
- カルーセル形式で選択
- 各キャラクターが動くアニメーション
- タップで選択、音声で名前読み上げ

### 4.3 カラースキーム
- **メインカラー**: 明るい青、黄色、緑
- **アクセントカラー**: オレンジ、ピンク
- **背景**: 白またはパステルカラー
- **高コントラスト**: 視認性を重視

### 4.4 アニメーション
- 滑らかで予測可能な動き
- 過度なアニメーションは避ける（集中力維持のため）
- ローディングも楽しい演出

## 5. 技術要件

### 5.1 推奨技術スタック（案）

#### フロントエンド
- **Framework**: Next.js 14+ (App Router)
- **言語**: TypeScript
- **3D表示**: Three.js + React Three Fiber
- **gLTF読み込み**: @react-three/drei
- **UI Components**: Tailwind CSS + Radix UI (アクセシビリティ対応)
- **アニメーション**: Framer Motion
- **ドラッグ&ドロップ**: dnd-kit

#### 状態管理
- **グローバル状態**: Zustand
- **サーバー状態**: TanStack Query

#### バックエンド（オプション）
- **API**: Next.js API Routes
- **データベース**: Supabase / Firebase
- **認証**: NextAuth.js

#### ホスティング
- **プラットフォーム**: Vercel
- **CDN**: Vercel Edge Network

### 5.2 非機能要件

#### 5.2.1 パフォーマンス
- 初回ロード: 3秒以内
- インタラクション応答: 100ms以内
- 60fps以上のアニメーション

#### 5.2.2 互換性
- **対応ブラウザ**: Chrome, Safari, Edge, Amazon Silk（最新版）
- **デバイス**: タブレット優先、PC対応
  - iPad（Safari）
  - Androidタブレット（Chrome）
  - Amazon Fire タブレット（Silk Browser）※低性能向け最適化対応
  - PC（Chrome, Safari, Edge）
- **画面サイズ**: 768px以上を推奨
- **Fire タブレット対応**:
  - 低負荷レンダリング設定（影、アンチエイリアス無効化）
  - 軽量gLTFモデル（Fire専用最適化版も検討）
  - タッチイベント最適化

#### 5.2.3 アクセシビリティ
- タッチ操作の最適化
- 音声ガイド対応
- 色覚特性への配慮（カラーユニバーサルデザイン）

#### 5.2.4 セキュリティ
- 個人情報の最小化
- 保護者による管理機能
- 広告なし、外部リンクなし

## 8. リスクと対策

### 8.1 想定リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 3D表示のパフォーマンス問題 | 高 | 軽量なgLTFモデル使用、LOD実装 |
| タッチ操作の難易度 | 高 | ユーザビリティテストを繰り返し実施 |
| 対象年齢に合わない難易度 | 中 | 段階的導入、調整可能な難易度設定 |
| ブラウザ互換性問題 | 中 | ターゲットブラウザを限定、事前テスト |
| 音声ガイドの品質 | 中 | プロの声優起用、複数パターン用意 |

### 8.2 技術的課題

- **gLTFモデルのファイルサイズ**: 圧縮と最適化で対応
- **ブラウザでの3D描画負荷**: WebGL対応確認、フォールバック用意
- **タッチイベントの精度**: デバイステスト環境の整備
