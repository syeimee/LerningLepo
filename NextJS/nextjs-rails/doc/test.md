# これで伝わる！エンジニアが言語化力を磨くためのステップ

- アウトプット

- 言語化

# はじめに


こんにちは、Watanabe Jin (<a href="/Sicut_study" class="user-mention js-hovercard" title="Sicut_study" data-hovercard-target-type="user" data-hovercard-target-name="Sicut_study">@Sicut_study</a>)です。<br>
突然ですが、あなたは人に自分の考えを伝えることは得意でしょうか？


エンジニアの方は職場によってはコミュニケーションをリアルに取ることがそもそもない。そんな方もそこそこいるかと思います。<br>
今回は最近注目されている<font color="blue"><strong>言語化</strong></font>について紹介してきます。<br>
言語化に関して書籍もたくさん出版されており、世間的にも注目度が高いものです。




<strong>私も言語化に魅了されている1人で4年にかけて言語化力を磨いてきました。</strong>


- 記事を600本以上投稿
- 音声配信を180日連続で投稿
- 100以上のアイデアから4つ起業
- LTに2ヶ月で13回登壇
おそらくエンジニアの中でも1位をあらそえるような活動量だと思っています。<br>
そんな言語化オタクの私が

「言語化力がつくメリット」<br>
「エンジニアにとっての言語化」<br>
「どうすれば身につくのか」<br>
を実際に私が言語化力をつけた経験を踏まえて紹介していきます。



この記事を読むこと「言語化」について理解して<font color="blue"><strong>身につける</strong></font>ことが可能です。<br>
<font color="red"><strong>言語化はあなたの人生を変える力を持っています</strong></font><br>
最後まで読んでいただき、実践していただけると嬉しいです。




# なぜ言語化力が大切なのか？


自分の伝えたいことがうまく伝えられずもやもやした経験はないでしょうか？
面接で質問されて頭が真っ白になった経験はないでしょうか？
<strong>このような経験がある人はかなり多いのではないかと思います。</strong>

これらは「言語化力」が大きく関係しています。<br>
<font color="red"><strong>言語化力とは頭にもやもやを言葉にすることです。</strong></font>



言語化が得意な人は、日頃からもやもやを言葉にする練習をしているので考えをうまく伝えられます。また頭の中で言葉が整理された状態になっているため、考えをしっかりと言語化して持っています。
<strong>面接で質問されてもすでに言語化されて整理されたことを話すだけなので真っ白になる心配もありません。</strong>


世の中には <strong>「頭の回転が早い」</strong> <strong>「話がうまい」</strong> と言われる人たちがいますが、すべての人が<font color="blue"><strong>言語化力</strong></font>を持っています。



私が4年間かけてトレーニングして感じるメリットは

- 面接が得意になる
- プレゼンテーションが得意になる
- 思ったことをしっかり伝えられる
- 話すのがうまいと言われる
- 文章をスラスラとかけるようになる
- コミュニケーション力が向上する
このようなものが特に身につけてよかったなと感じます。これらのスキルは人が生きていく上でも、仕事をしていく上でもできるようになる方がお得なスキルです。

人前で話せるほうがチャンスが回ってきます。<br>
面接が得意なら転職の多いエンジニア業界でも少ない面接で通過できます。<br>
文章がスラスラかければ記事を書いてアウトプットすることも習慣化できます。


<font color="red"><strong>言語化力を1度身につけてしまえば複利的に効果をもたらしてくれます</strong></font>

# 言語化力を身につけるには？


メリットばかりの言語化ですが、<font color="blue"><strong>身につけるのはかなり難しいスキル</strong></font>です。<br>
だから多くの書籍が続々と販売されているのだと思います。そしてどれも似通った内容です。


<strong>言語化力は才能を持っている人はほとんどおらず、練習で身につけるものです。</strong><br>
裏を返せば、<font color="blue"><strong>練習しない限り身につけることができないスキル</strong></font>なのです。<br>
世の中の殆どの人が「言語化」の練習をしたことがないはずです。練習をしたことがあったとしても「継続」している人はほとんどいません。<br>
<br>






<font color="red"><strong>言語化が身につくには大量の練習が必要になります。</strong></font><br>
一朝一夕の練習だけでは身につくことは絶対にないのです。<br>
<strong>だからこそ、言語化力を身につけることができればあなたの武器になるのです</strong>





このあとに実際に私がやって効果のあったトレーニング方法を紹介しますが、全てに共通しているのは　<strong>「何か感じたことをスルーするのではなく、気づいて言葉にする練習」</strong>　ということです。<br>
<br>



人には <strong>「意識」と「無意識」</strong> が存在しており、無意識の割合が95%とほとんどが無意識で行われています。車を運転するときに、教習所で練習していたときのようにビクビクしながら運転することはないでしょう。むしろ、家族と話しながら運転することすら余裕なはずです (私はゴールド免許ですがペーパーなので無理ですが)


人は <font color="blue"><strong>「何かを感じた」</strong></font>ときに、無意識的に感じて終わります。<br>
「何かを感じた」への解像度がかなり低いと無意識で終わってしまいます。



<font color="red"><strong>その「何かを感じた」瞬間に意識的に言葉にすることで解像度が高まり、言語化できるようになるのです</strong></font><br>
<br>



「いや、何かを感じることなんてないよ」<br>
<br>


そんな声が聞こえますが、<br>
「映画をみて感動した」<br>
「SNSをみて思わず投稿をみてしまった」<br>
「なんかいいなと思った」



<strong>このような瞬間は言語化をする絶好の機会なのです。</strong>

言語化をしようと思うと最初はうまく言葉にするのは難しいです。<br>
しかし、小さな言葉にすることができるとそこから連鎖的に言葉にできるようになります。また一度言語化をすると生活の中でどんどん言語化が深まるようになり、体系的になっていきます。

<strong>これがあなたの意見や考えになり、面接で聞かれてもすぐに答えられることに繋がります。</strong>


言語化をしたことを伝えることは、<font color="red"><strong>聞いている人が言語化していないことを代弁することにつながることもあります。</strong></font><br>
すると「自分も同じことを考えていた」というような「気づき」にも繋がります。


# 言語化力を身につける方法


言語化力を身につける方法は世の中にいくらでもあります。<br>
「何か感じたことをスルーするのではなく、気づいて言葉にする練習」をすればそれは言語化のトレーニングです。

ここでは私が特に効果があったと思う方法をいくつか紹介していきます。<br>
どの方法にしても言語化を身につけるには*<font color="red"><strong>継続的なトレーニングが必須</strong></font>です。


時間がかかって地道だからこそ身につけたときに <strong><font color="blue">「武器」</font></strong> になるのです。<br>
この方法をどれか選んで<strong>3ヶ月実践すれば確実に成長できる</strong>ので、行動してみてください。



## 1. 0秒思考


皆さんは<font color="blue"><strong>「0秒思考」</strong></font>聞いたことありますか？<br>
0秒思考は以下の本で提唱されている言語化トレーニングとなります。


<iframe id="qiita-embed-content__6cbfaec35671b56838eb0f056c853118" src="https://qiita.com/embed-contents/link-card#qiita-embed-content__6cbfaec35671b56838eb0f056c853118" data-content="https%3A%2F%2Famzn.asia%2Fd%2FgqyPVRE" frameborder="0" scrolling="no" loading="lazy" style="width:100%;" height="29">
</iframe>

この方法は短時間で終わり、言語化力が伸びることはもちろん色々メリットがあるので有名な方法です。
- A4用紙を用意します
- 左上にタイトル、右上に日付を書きます
- テーマを決めます。例えば「私の叶えたい夢」など
- 1分程度でテーマについて考えて浮かんだ言葉をとにかくメモします
- 1-4を10回繰り返します
テーマについて1分間考えてとにかく言語化していくというのは、最初かなり難しいです。<br>
その難しさこそが言語化のトレーニングになります。<br>
また、朝イチに0秒思考をやることで脳内を整理することができて1日をいい気分で始められます。


私が運営する<a href="https://projisou.jp" rel="nofollow noopener" target="_blank">プログラミングコーチングJISOU</a>でも朝活会や合宿などでも朝に必ず0秒思考を取り入れています。

言語化トレーニングという面もありますが、<font color="red"><strong>目標への解像度を高める</strong></font>ために行っています。<br>
目標の解像度が低いと叶えるのは不可能です。<br>
自分が目指す先がしっかりと描けていることは目標を達成する上で最初にやるべきことです。<br>
朝イチで目標を再認識することでモチベーションを高めることにもつながります。




<strong>0秒思考は1日10分程度で気軽にでき、メリットも多いので初心者におすすめの言語化トレーニングです。</strong>

## 2. 記事を書く


私が言語化スキルを身に着けたのは<font color="blue"><strong>記事を書く習慣</strong></font>があったからです。<br>
いまではQiitaに600本近い記事を投稿しています。ただ注意が必要なのは <strong><font color="blue">記事を書く種類によっても効果が変わる</font></strong> ということです。




記事を書くときに大きく2種類のテーマがあると考えます。
- エラー解決型記事
- 経験や体験を伝える記事
これらの記事の考え方については以前詳しく紹介しています👇

ここで注目してほしいのは、<font color="red"><strong>「エラー解決の記事は事実を書いていくだけなので、思ったことを言語化するプロセスがすくない」</strong></font> ということです。<br>
言語化力をもし上げたいと考えるのであれば、あなたの経験や体験、成功、失敗からの気づきや学びをシェアするような記事を書いていくのがおすすめです。



エンジニアは技術に関する記事が多くなりますが、技術に関する情報は必ず公式ドキュメントなど<font color="blue"><strong>1次情報</strong></font>が存在します。つまり、Qiitaを見なくても最低限なんとかなります。いまではAIが教えてくれるので余計にそう感じてしまいます。


<font color="red"><strong>しかし、「経験」「体験」「成功」「失敗」などは誰かが記事に書かない限り増えることはありません。</strong></font><br>
誰かの失敗を学び、失敗を回避することは成功方法を学ぶよりも確実に成功へと近づきます。誰かのやったうまくいった勉強法を学ぶことは、非効率な学習をして時間を失うことを防ぎます。



実際に私も<font color="blue"><strong>得意なアウトプットに関する記事</strong></font>を出すと多くの人が参考にしてくれています。言語化する人がいない分、そのような記事には価値が生まれているのです。

人に気づきを与えて、自分の言語化力も向上するので「記事を書く」はおすすめです。<br>
必ず言語化力が確実に向上することをお約束します。

## 3. 気づきを音声配信する


私は<a href="https://stand.fm/channels/65db27e353b400abe28ec29a" rel="nofollow noopener" target="_blank">Stand.fm</a>という音声配信SNSで5分の投稿を投稿を半年間連続で行いました。

<iframe id="qiita-embed-content__0f84f40c2f4ab2495c447d7f7b2bf60f" src="https://qiita.com/embed-contents/link-card#qiita-embed-content__0f84f40c2f4ab2495c447d7f7b2bf60f" data-content="https%3A%2F%2Fstand.fm%2Fchannels%2F65db27e353b400abe28ec29a" frameborder="0" scrolling="no" loading="lazy" style="width:100%;" height="29">
</iframe>

毎日投稿し始めるとわかりますが、<strong>話す話題がなくなっていきます。</strong><br>
毎日毎日、「話す話題がない」と感じて過ごしていると、日常に起きる <font color="blue"><strong>小さな気付き</strong></font> が発信の内容になっていきました。日頃から話す話題を見つけることでスルーしがちな「気づき」をどんどん言語化して連鎖的につなげることで体系化することもできるようになりました。




また<font color="blue"><strong>音声配信</strong></font>をすることは、他にもメリットがあります。<br>
あとからわかりましたが、頭の中が言語化で整理されていたとしても <font color="blue"><strong>「声」</strong></font>に出さないと言語化が本当にできているかがわからないということがあります。「人にわかりやすく説明する」ということを通して言語化をより深めることができるのです。



JISOUの中ではこの方法を活用して転職をするときに1日5分を2回、<font color="red"><strong>「過去の経歴ややってきたことについて1つずつ話す」</strong></font>という練習をさせています。

この練習をすることで、1つ1つの面接で聞かれそうなことに対して相手に気づきを与えられるように話す練習であったり、5分間話すために言語化をより深めることが可能になります。
<font color="red"><strong>言語化力を身につけるなら声にだして人に説明することは重要です。</strong></font>

## 4. 読書をする


皆さんは読書を普段していますでしょうか？<br>
私は読書が好きで(技術書ではない)、月に5冊ほど読んでいます。

なぜ本を読むのかというと、<font color="red"><strong>自分の言語化に深みをもたせたいから</strong></font>というのがあります。<br>
なので必然的に読む本は自分がより多くの知見を得たいジャンルの本になっていきます。


書籍を「言語化」のために読むと自分と同じ意見であったり、反対意見、気づいてもいなかった視点での気付きなど多くの発見があります。ここで重要なのが<strong>気づいていなかった視点</strong>です。これはまさに誰かが自分の代わりに言語化をしてくれていたものです。

<font color="red"><strong>言語化は必ずしも自分でする必要はなく、時には誰かの言語化によって自分の言語化が深まることがあるのです。</strong></font>

# おわりに


今回は言語化力を身につけることの大切さと、その方法について実体験をもとに紹介しました。
書籍に乗っているものから、完全オリジナルの方法まで様々でしたが、<strong>すべての行動にはエンジニアとしてのメリットがあり、その結果「言語化力」が身につけられるようになっています。</strong>


言語化力は一朝一夕では身につかず、地道な練習が必要です。しかし、エンジニアとしての価値を高める活動の中で十分身につけられるのでとてもお得だと考えています。<br>
ぜひとも普段の活動を意識して、<strong>言語化力のあるエンジニア</strong>を目指していただけたらと思います。
